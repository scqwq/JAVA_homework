# 学生宿舍管理分配系统项目规划

本项目面向学校宿舍管理场景，目标是完成学生信息管理、宿舍楼与房间管理、住宿分配、调宿以及宿舍查询等核心功能。规划时首先考虑到该系统实体数量不多，但实体之间关系明确，且数据一致性要求较高，因此重点不在复杂界面，而在业务规则和数据库约束的设计。

技术上，本项目选择 `Java 21 + JDBC + Maven + 内置 HttpServer` 的轻量方案，而没有直接使用 Spring Boot。这样选择的原因是项目规模较小，核心任务是体现面向对象建模、数据库设计和分层思想，轻量方案更有利于突出业务本身，也便于部署到云服务器。

模型设计上，系统以 `Student`、`Building`、`Room` 和 `DormAssignment` 为核心对象，并补充 `Gender`、`BuildingGenderPolicy` 等枚举或规则模型，用来表达学生性别和宿舍楼入住策略。数据库采用关系型结构，主要包括 `students`、`buildings`、`rooms`、`dorm_assignments` 四张表，通过主键、外键和唯一约束保证“一个学生只能有一条住宿记录”“同一床位不能重复分配”等关键规则。

业务层采用分层设计，将系统划分为 `Handler`、`Service`、`Repository`、`Model` 四层。其中 `Service` 层负责处理学生存在性、楼栋性别限制、房间归属和床位冲突等核心逻辑，避免业务判断分散在页面或 SQL 中，提高系统的可维护性。

前端方面，项目采用服务端直接生成 HTML 的方式，实现一个集中式管理页面，便于完成录入、分配、调宿和查询操作。部署上，系统可运行在华为云 ECS 服务器中，数据库可使用 PostgreSQL 或 MySQL，既能满足课程作业演示需求，也具备后续扩展和上线的基础。
